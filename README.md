1. Abstract

This project presents the design and development of a web-based Early Childcare Staff Wellbeing Management Platform, aimed at addressing the increasing challenges associated with staff wellbeing in early childhood education environments.

A key feature of this system is the implementation of anonymous wellbeing checks, ensuring that staff can provide honest feedback without fear of identification or repercussion. This design choice supports more accurate data collection and aligns with ethical standards in workplace monitoring systems.

2. Introduction

Staff wellbeing is a critical factor influencing both employee retention and childcare quality outcomes. Early childcare professionals frequently experience high levels of stress due to workload, emotional demands, and organisational pressures.

Traditional management systems often fail to adequately capture the true wellbeing state of staff, particularly when anonymity is not guaranteed.

This platform addresses this gap by providing:

Anonymous wellbeing reporting mechanisms
Centralised monitoring tools for management
Data-informed insights to support decision-making

3. Aims and Objectives
Aim

To design and implement a functional prototype of a wellbeing management system that prioritises anonymity, accessibility, and usability.

Objectives
Develop a user-friendly wellbeing tracking interface
Ensure anonymous submission of staff wellbeing data
Provide visual representation of wellbeing trends
Enable effective managerial oversight without compromising privacy

4. System Design and Architecture

The platform adopts a front-end web architecture, prioritising simplicity and accessibility.

Technologies Used
HTML5
CSS3
JavaScript
GitHub Pages

Design Considerations
Anonymity by design (no personally identifiable data collected when completeing wellbeing check ins)
Managers will be able to see employee information when completing incident reports for safety standards 
Ease of use for non-technical childcare staff
Low system requirements (browser-based access)
Future scalability for backend integration

5. Implementation

The system is implemented as a static web application with interactive components powered by JavaScript.

Key features include:

Anonymous wellbeing input interface
Visual feedback and simple analytics
Clean and structured user interface

The absence of user authentication is a deliberate design decision to preserve full anonymity, encouraging honest and unbiased staff responses.

6. Evaluation and Discussion
Strengths
Promotes honest reporting through anonymity
Highly accessible (no installation required)
Simple and intuitive interface
Addresses a real-world organisational challenge

Limitations
No persistent data storage
Limited scalability without backend integration
Basic analytics functionality

7. Future Work

Future development could enhance both functionality and security:

Secure backend integration with optional anonymisation controls
Advanced analytics dashboards
Mobile optimisation
Compliance with data protection regulations (e.g., GDPR)
Configurable anonymity settings for different organisational needs

8. Ethical and Professional Considerations

This system is designed with strong emphasis on ethical data handling.

The use of anonymous wellbeing checks ensures:

Protection of staff identity
Reduction of reporting bias
Increased trust in the system

Future iterations must maintain:

Data confidentiality
Transparency in data usage
Compliance with relevant data protection frameworks
9. Conclusion

This project demonstrates the effectiveness of a lightweight, web-based solution in addressing staff wellbeing challenges within early childcare settings.

By embedding anonymity as a core design principle, the platform improves the reliability of collected data and supports more informed organisational decision-making.
